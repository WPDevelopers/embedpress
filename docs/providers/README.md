# Provider System

The Provider system is the heart of EmbedPress. Every embed in the plugin is generated by a Provider class.

## What a Provider is

A **Provider** is a PHP class that knows how to:

1. Recognize that a URL belongs to it (via host pattern matching).
2. Parse the URL into the parts that matter (video ID, file ID, channel handle…).
3. Generate the embed HTML — usually an iframe pointing at the upstream embed endpoint.

Each Provider lives in `EmbedPress/Providers/<Name>.php` and is mapped to one or more host patterns in `providers.php`.

## Why the adapter pattern

EmbedPress originally wrapped the [Embera](https://github.com/mauricius/embera) library, which itself uses an adapter pattern: one adapter per service. We kept that pattern even as we added our own providers because:

- **Isolation**: a bug in the YouTube adapter can't break Vimeo.
- **Testability**: each adapter is a small unit you can mock.
- **Clarity**: adding a new source means adding one file, not hunting through a giant switch.

## Anatomy of a Provider

Most providers extend `Embera\Adapters\Service`. The minimum interface is:

```php
namespace EmbedPress\Providers;

use Embera\Adapters\Service as EmberaService;

class ExampleProvider extends EmberaService
{
    /** Validate this URL belongs here */
    public function validateUrl(\Embera\Url $url)
    {
        return preg_match('~example\.com/watch/[A-Za-z0-9_-]+~', (string) $url);
    }

    /** Normalize URL into a canonical embed URL */
    public function normalizeUrl(\Embera\Url $url)
    {
        return $url;
    }

    /** Build the iframe HTML */
    protected function modifyResponse(array $response = [])
    {
        $response['html'] = sprintf(
            '<iframe src="%s" width="%d" height="%d" frameborder="0" allowfullscreen></iframe>',
            esc_url($this->getEmbedUrl()),
            $this->getWidth(),
            $this->getHeight()
        );
        return $response;
    }

    private function getEmbedUrl()
    {
        // …extract ID, build embed URL…
    }
}
```

The `Wrapper` class (`EmbedPress/Providers/Wrapper.php`) is a permissive base used as the **fallback provider** when no specific provider matches. See [Wrapper](../features/wrapper.md).

## How URL → Provider routing happens

1. User-supplied URL hits `Core::parseContent()`.
2. Core constructs an Embera engine seeded with `$additionalServiceProviders` from `providers.php`.
3. Embera iterates registered providers and asks each `validateUrl($url)`.
4. The first provider that says "yes" wins. If none match, the **Wrapper** provider takes the URL (last-resort fallback).
5. Core calls the matched provider, captures its HTML, and forwards it through the [Feature_Enhancer](../features/feature-enhancer.md) filter pipeline.

> **Greediness warning.** A provider with a too-broad host pattern can steal URLs from neighbors. Always test that your new provider doesn't match `youtube.com/...` or `vimeo.com/...` URLs, and run the E2E provider suite before merging.

## The provider registry: `providers.php`

`providers.php` is a flat associative array:

```php
$additionalServiceProviders = [
    EMBEDPRESS_NAMESPACE . "\\Providers\\Youtube"
        => ["youtube.com"],
    EMBEDPRESS_NAMESPACE . "\\Providers\\Wistia"
        => ["*.wistia.com", "wistia.com"],
    // …
];
```

The wildcard `*.foo.com` matches any subdomain. EmbedPress also extends Embera's matching to support the wildcard.

`SelfHosted` is special: its host list is the *current site* plus a long list of TLDs (`*.com`, `*.net`, …) so any direct `.mp4` / `.mp3` / `.pdf` URL can be picked up and routed to the self-hosted media renderer.

## What "providers" do *not* do

- They don't render UI controls (that's the editor's job).
- They don't decide whether to apply the Custom Player (Feature_Enhancer does that based on attributes).
- They don't enqueue assets (Asset Manager does, when a marker class is detected).
- They don't talk to any analytics or third-party APIs unless that's literally how the provider embeds (e.g., Google Photos requires a fetch to derive the album HTML).

A Provider is a **pure URL → HTML transformer**. Keep it that way.
